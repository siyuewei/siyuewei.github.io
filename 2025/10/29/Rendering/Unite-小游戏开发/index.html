移动平台的图形引擎

概述

WebGL和 WebGPU 是用于网页浏览器的图形 API，Metal 和 Vulkan 是用于操作系统和硬件的“原生” (Native) 图形 API。

WebGL 和WebGPU是网页渲染的写法，但是真实渲染到电脑上，需要用电脑的渲染API，所以要翻译。而翻译这件事是浏览器在做的。

WebGL

WebGL概述

  • 定义：JavaScript API，用于在浏览器中渲染 2D 和 3D 图形，无需任何插件。
  • 版本历史：
    • WebGL 1.0 基于 OpenGL ES 2.0
    • WebGL 2.0（目前主流）基于 OpenGL ES 3.0（2012年发布）
  • 现状：已经过时，WebGL不会再有任何Major Feature Update。
  • 主要限制：缺失现代图形API的常用特性：Compute Shader，SSBO，Indirect Draw

2. SSBO（Shader Storage Buffer Object，着色器存储缓冲对象）

什么是SSBO？

  • 一种GPU缓冲区类型，允许着色器读写大量结构化数据。
  • 是UBO（Uniform Buffer Object）的增强版本。

SSBO vs UBO对比：

特性 UBO SSBO
访问权限 只读 可读写
大小限制 通常16KB-64KB 可达GB级别
数据对齐 严格的std140布局 更灵活的std430布局
性能 读取快 读写都支持
用途 传递常量数据 动态数据、计算结果

主要优势：

  • 可写性：着色器可以修改数据，实现GPU端的数据更新
  • 大容量：可以存储海量数据（如百万级粒子）
  • 灵活结构：支持动态数组、复杂数据结构
  • 原子操作：支持原子计数器、原子交换等并发操作

典型应用场景：

  • GPU粒子系统（存储粒子状态）
  • GPU Skinning（骨骼动画数据）
  • Indirect Draw的参数缓冲
  • 光照计算的Tile数据
  • 自定义渲染管线的中间结果存储

3. Indirect Draw（间接绘制）

什么是Indirect Draw？

  • 一种绘制调用方式，允许GPU自己决定绘制参数，而不需要CPU每次都传递参数。
  • 绘制参数（顶点数量、实例数量等）存储在GPU缓冲区中。

主要优势：

  • 减少CPU-GPU同步：绘制参数在GPU侧生成，无需回读到CPU
  • GPU驱动渲染：结合Compute Shader可以实现完全GPU驱动的渲染管线
  • 动态LOD：GPU可以根据距离动态调整细节级别
  • 高效剔除：GPU剔除后直接修改绘制参数，不可见物体instanceCount设为0

典型应用场景：

  • GPU Driven Rendering：整个渲染决策在GPU侧完成
  • 大规模场景渲染：草地、树木、建筑的批量渲染
  • 动态剔除：视锥剔除、遮挡剔除后动态调整绘制
  • 粒子系统：GPU计算存活粒子数量并绘制
  • 程序化实例化：GPU生成实例化参数

WebGL的替代方案

由于WebGL缺失这些现代特性，现代浏览器正在推进新的图形API：

  • WebGPU：
    • 基于现代图形API（Vulkan、Metal、DirectX 12）
    • 支持Compute Shader、SSBO、Indirect Draw等所有现代特性
    • 更好的性能和更低的开销
    • Chrome、Firefox、Safari已开始支持

小结

这三个特性共同构成了现代GPU驱动渲染管线的基础：

  1. Compute Shader：提供通用计算能力
  2. SSBO：提供大容量可读写存储
  3. Indirect Draw：实现GPU驱动的绘制决策

它们的组合使用可以实现：

  • 将渲染决策从CPU转移到GPU
  • 减少CPU-GPU同步开销
  • 实现更复杂的渲染算法
  • 处理更大规模的场景

WebGL因为基于老旧的OpenGL ES 3.0，无法支持这些特性，这也是为什么它在现代图形应用中逐渐被WebGPU取代。
